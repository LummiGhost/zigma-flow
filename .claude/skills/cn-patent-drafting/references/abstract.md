# 说明书摘要撰写规范
> 依据：专利法第26条第1款；专利法实施细则第23条；《专利审查指南》第一部分第一章

## 基本要求

- **字数限制**：不超过 **300 字**（中文）
- **法律地位**：摘要仅为技术信息，**不具有法律效力**，不能用于解释权利要求范围，不能作为申请日后修改的依据
- **核心功能**：便于公众快速获取专利的技术要点，用于信息检索

## 必须包含的四要素

1. **发明名称**（或实用新型名称）
2. **所属技术领域**
3. **技术问题**（要解决的问题）
4. **技术方案要点**（核心技术特征，对应独立权利要求）
5. **主要用途**（应用场景）

## 撰写模板

```
本[发明/实用新型]涉及[技术领域]，具体涉及[具体方向]。

针对[现有技术存在的问题]，本[发明/实用新型]提供了一种[名称]。

[技术方案要点描述，150字左右：核心结构/步骤/特征]。

本[发明/实用新型]能够[主要有益效果/用途]。
```

## 示例

```
本发明涉及图像检测技术领域，具体涉及一种基于深度学习的焊缝缺陷检测方法及装置。

针对现有人工检测效率低、漏检率高的问题，本发明提供了一种自动化检测方案。

该方法包括：获取焊缝图像；将图像输入特征提取网络提取多尺度特征；将多尺度特征输入缺陷分类器输出缺陷类型和置信度；根据置信度与预设阈值比较输出检测结论。其中，特征提取网络采用改进的ResNet结构，在浅层和深层特征之间引入跨层连接以增强小缺陷的检测能力。

本发明可用于工业焊缝质量在线检测，检测准确率达98%以上，较人工检测效率提升10倍。
```

## 禁忌

- ❌ 超过 300 字
- ❌ 使用宣传性语言（"最优"、"突破性"）
- ❌ 仅有法律条文式表述而缺少技术内容
- ❌ 将权利要求书全文复制为摘要
- ❌ 包含商业价值评价

## 摘要附图

- 若申请含有附图，需指定一幅最能说明发明技术特征的图作为摘要附图
- 摘要附图通常为最重要的结构图或流程图（如图1）

# 软件/计算机程序相关专利注意事项
> 依据：《专利审查指南》第二部分第九章

## 可申请专利的计算机程序相关发明

中国专利法对软件专利的保护采用"技术性"判断标准：**必须涉及技术手段解决技术问题，产生技术效果**。

### 可获得保护的主题
- ✅ 涉及计算机内部运行性能改善的程序（如内存管理优化）
- ✅ 与外部技术处理的结合（图像/信号处理、控制系统）
- ✅ 计算机实现的方法（有明确技术问题和技术效果）
- ✅ 包含程序的计算机可读存储介质
- ✅ 包含程序的计算机系统/装置

### 不可获得保护的主题
- ❌ 纯商业规则/游戏规则（无技术特征）
- ❌ 纯数学方法
- ❌ 单纯的数据呈现/界面美化（无技术问题）

## 软件专利的权利要求布局建议

对于一个AI/软件发明，建议同时撰写：

```
权利要求1：方法权利要求（步骤描述）
权利要求2-N：方法从属权利要求
权利要求N+1：装置/系统权利要求（与方法对应的模块化结构）
权利要求N+2-M：装置从属权利要求
权利要求M+1：计算机可读存储介质（程序产品保护）
```

## 说明书中需强调的技术性

在背景技术和发明内容中，应突出：
- 具体的**技术问题**（如"现有方法的计算复杂度为O(n²)，无法实时处理"）
- 具体的**技术手段**（如"采用注意力机制减少特征图冗余计算"）
- 可测量的**技术效果**（如"将推理时间从200ms降低至20ms"）

避免仅描述商业效果（如"提升用户满意度"、"增加营收"）。

## 算法描述规范

在具体实施方式中描述算法时：
1. 可配合流程图说明处理步骤
2. 对关键数学公式使用规范的数学表达
3. 神经网络结构需说明：层数、连接方式、激活函数、训练方法

示例：
```
[0023] 所述特征提取网络采用改进的ResNet-50结构，具体包括：
输入层接收224×224×3的图像张量；
4个残差块组，分别包含3、4、6、3个残差单元；
在第2残差块组和第3残差块组之间引入跨层连接，
将第2残差块组输出的特征图经1×1卷积调整通道数后与第3残差块组的输出进行逐元素相加，
以融合浅层细节特征和深层语义特征。
```
