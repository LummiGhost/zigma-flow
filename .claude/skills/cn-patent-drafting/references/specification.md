## 技术领域

### 要求
- 明确指出发明或实用新型所属或直接应用的技术领域
- 使用规范的技术术语，与 IPC 分类一致

### 写法模板
```
本发明涉及[大领域]技术领域，具体涉及[具体子领域]。
```

### 示例
```
本发明涉及人工智能技术领域，具体涉及一种基于深度学习的图像缺陷检测方法及系统。
```

### 常见错误
- ❌ "本发明涉及一种先进的、革命性的技术" → 含宣传性词语
- ❌ 技术领域过于宽泛（"本发明涉及电子技术"）
- ❌ 直接写产品名称而非技术领域

## 背景技术

### 要求
- 介绍与发明最接近的现有技术
- 客观分析现有技术存在的缺点或不足
- 可以引用专利文献或非专利文献（注明出处）
- 避免过度贬低现有技术（可能引发不必要的法律争议）

### 结构
1. 现有技术描述（什么技术已经存在）
2. 现有技术不足（存在什么问题/缺陷）

### 写法模板
```
[现有技术方案描述]。

然而，上述[现有技术]存在以下不足：[缺陷1]；[缺陷2]……
```

### 技术文献引用格式
```
参见专利文献CN XXXXXXXX A（申请人名，公开日期）。
参见非专利文献：作者，"文章名"，期刊名，年份，卷（期）：页码。
```

### 常见错误
- ❌ 仅泛泛描述技术背景，不分析具体缺陷
- ❌ 使用"该技术毫无价值"等过激评价
- ❌ 背景技术与发明实际解决的问题脱节

## 发明内容

### 三个必须包含的子部分

#### 2.1 技术问题
- 针对背景技术的缺陷，提出本发明要解决的具体技术问题
- 与背景技术的"缺陷"一一对应

**模板**：
```
为了解决上述[背景技术中的缺陷描述]的技术问题，本发明提供了一种[发明名称]。
```

#### 2.2 技术方案
- **必须与权利要求书保持一致**，通常直接引述权利要求
- 独立权利要求对应的技术方案是核心
- 从属权利要求对应的优选方案在此说明

**模板**：
```
本发明的技术方案如下：

一种[名称]，包括/其特征在于：
[特征1]；
[特征2]；
……

进一步地，[优选特征说明]。
```

> ⚠️ 注意：技术方案的表述措辞应与权利要求书中保持一致，避免出现说明书有而权利要求没有的技术特征导致解释冲突。

#### 2.3 有益效果
- 说明本发明相对于现有技术具有的优点和积极效果
- 有益效果应能从技术方案中直接导出，不得无中生有
- 可附数据支撑（如"测试表明，本发明使效率提升30%"）

**模板**：
```
本发明具有以下有益效果：
（1）[效果1]，原因在于[对应技术特征]；
（2）[效果2]；
……
```

## 附图说明

### 要求
- 仅当申请包含附图时才需要此部分
- 简要说明每幅附图的图名/内容
- 每幅附图须编号，如"图1"、"图2"

### 格式
```
图1为本发明实施例提供的[装置名称]的结构示意图；
图2为本发明实施例提供的[方法名称]的流程图；
图3为本发明实施例中[具体模块]的详细结构图。
```

### 附图标记说明（可选，推荐）
```
其中：
1-[部件名称]；2-[部件名称]；3-[部件名称]；
10-[模块名称]；20-[模块名称]。
```

## 具体实施方式

### 要求（最重要部分之一）
- 以所属技术领域的技术人员能够实现为标准
- 至少包含**一个优选实施例**（建议2-3个，覆盖不同场景）
- 实施例应支持独立权利要求的最宽保护范围
- 结合附图详细说明（若有附图）

### 实施例结构
```
实施例一：[简短名称，如"一种XX装置的具体实施方式"]

参见图1，[整体描述]。

[组成部分1]为……，用于……
[组成部分2]为……，其中……

本实施例的工作原理/工作过程如下：……

本实施例的有益效果：……

实施例二：[覆盖权利要求的另一种实现方式]
……
```

### 数值范围的处理
- 说明书中给出的具体数值，会在审查时被认为是公开内容
- 若权利要求中有数值范围，说明书的实施例数值应落在该范围内
- 建议在实施例中给出范围两端及中间值的实施例

### 功能性特征的处理
- 如权利要求中使用功能性限定（"用于…的装置"），说明书中**必须**有对应的具体结构实施例
- 功能性限定的保护范围 = 说明书中记载的实施方式及其等同

## 通用格式规范

### 段落标号
- 每个自然段应有四位数字段号，如 `[0001]`、`[0002]`
- XML格式提交时段号格式：`<p num="0001">`

### 书写规范
- 页面：A4纵向
- 中文撰写（外来词可附英文括注）
- 不使用"我"、"我们"、"申请人认为"等主观表述
- 技术术语应贯穿全文保持一致

### 禁止内容
- ❌ 宣传性词语（"最先进"、"最优秀"、"革命性"）
- ❌ 水印、宏命令、嵌入对象、超链接
- ❌ 对现有技术的过度否定

## 充分公开要求

根据专利法第26条第3款，说明书**必须**达到"所属技术领域的技术人员能够实现"的程度。

### 充分公开的判断标准
领域技术人员（具备该技术领域普通知识、了解申请日前公知常识的虚拟人）能否：
1. 理解发明的技术方案
2. 再现该发明（即实施该发明）
3. 解决说明书所声称解决的技术问题

### 常见公开不充分情形
| 情形 | 说明 |
|------|------|
| 关键步骤缺失 | 方法权利要求中某步骤的操作条件未说明 |
| 实施例与权利要求脱节 | 独立权利要求范围宽，实施例仅覆盖一个极窄场景 |
| 化学/生物无实验数据 | 医药领域无体内/体外实验支持技术效果 |
| 功能性特征无结构支撑 | "用于处理X的模块"但未说明如何处理 |
