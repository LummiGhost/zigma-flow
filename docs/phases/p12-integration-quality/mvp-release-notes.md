# MVP Release Candidate Notes

Issue: #85 / P12.4 MVP release candidate

Date: 2026-06-23

## Release Candidate Scope

This release candidate packages the MVP local workflow runtime as a single-process TypeScript CLI. It is scoped to the MVP contracts in `docs/mvp-contracts.md` and does not introduce remote execution, web UI, PR automation, or multi-agent orchestration.

Included MVP surfaces:

- Local project initialization with `.zigma-flow/` workflows, skills, config, and lockfile assets.
- Workflow validation for MVP workflow and skill-pack definitions.
- Run creation, active run tracking, status rendering, prompt generation, step reports, signals, and deterministic gates.
- Code-change dogfood workflow assets generated by `zigma-flow init`.
- Artifact and event persistence for auditability.
- Router, script, check, and read-only workspace guard behavior covered by tests.

## P12.4 Release Fixes

- `zigma-flow run <workflow-name>` now starts initialized workflows by name, for example `zigma-flow run code-change --task "..."`.
- `zigma-flow status` now keeps job states readable when a job id is longer than the default status column.
- README and P12 docs now use the canonical report path and the actual default static/unit check commands.
- P12 dogfood docs now record the P12.3/P12.4 source fixes rather than leaving stale "pending fix" notes.

## Validation Summary

Release-candidate validation covers:

- TypeScript typecheck.
- Repository lint script.
- Unit, integration, E2E, and full Vitest suites.
- Build and CLI smoke.
- Real init, validate, run, status, and prompt dogfood against the generated `.zigma-flow/` assets.

Detailed command evidence is in `docs/phases/p12-integration-quality/mvp-release-verification-log.md`.

## Known Non-Blocking Risks

- The release tag is not cut by this PR. Tagging should happen only after merge and green main-branch CI.
- The repository `lint` script currently aliases `tsc --noEmit`; there is no separate style linter gate yet.
- `npm link` / packaged install smoke remains tracked as TD-P12-001. The MVP release candidate is validated through the built `dist/cli.js` CLI path.
- CI is authoritative for merge readiness; local gates do not replace `gh pr checks <PR>`.

## Out of Scope

The following remain intentionally outside this MVP release candidate:

- Remote Skill Registry.
- Runtime YAML patching.
- Arbitrary loops or a general expression language.
- Automatic multi-agent concurrent scheduling.
- Docker sandbox.
- MCP runtime.
- PR automation.
- Automatic Issue or Project creation.
- Web UI.
- Multi-tenant permission platform.
- Complex LLM Judge.
- Full event-sourcing rebuild.
- Full Zigma OS distribution.
