## 类型与结构

### 按保护客体分类

| 类型 | 适用场景 | 示例 |
|------|----------|------|
| 产品权利要求 | 装置、设备、系统、物品、组合物 | 一种图像处理系统 |
| 方法权利要求 | 制造方法、使用方法、通信方法 | 一种图像处理方法 |
| 用途权利要求 | 已知产品的新用途 | XX化合物在制备YY药物中的应用 |

### 按层级分类
- **独立权利要求**：包含实现发明目的的全部必要技术特征，不引用其他权利要求
- **从属权利要求**：引用在前权利要求，增加附加技术特征进行进一步限定

### 权利要求书整体结构
```
权利要求1（独立，产品/方法）
权利要求2（从属于1，优选A）
权利要求3（从属于1，优选B）
权利要求4（从属于2，更具体）
权利要求5（独立，另一维度，如方法/装置）
……
```

## 独立权利要求

### 核心原则
**必要技术特征原则**：独立权利要求应当且只应当包含解决技术问题所必不可少的技术特征。

- 删除任何非必要特征 → 扩大保护范围
- 确保技术方案完整 → 缺少必要特征则权利要求不能独立实现发明目的

### 两段式结构（推荐用于有现有技术基础的发明）
```
一种[发明名称]，包括[现有技术已有特征]，
其特征在于，[区别技术特征1]，[区别技术特征2]。
```

### 一段式结构（全新技术方案，无法划分）
```
一种[发明名称]，其特征在于，包括：
[技术特征1]；
[技术特征2]；
[技术特征3]。
```

### 产品独立权利要求示例
```
一种基于深度学习的焊缝缺陷检测装置，包括图像采集模块和处理器，
其特征在于，还包括：
特征提取网络，用于从所述图像采集模块获取的焊缝图像中提取多尺度特征；
缺陷分类器，与所述特征提取网络连接，基于所述多尺度特征输出缺陷类型和置信度；
阈值判决单元，根据所述置信度与预设阈值的比较结果输出检测结论。
```

### 方法独立权利要求示例
```
一种基于深度学习的焊缝缺陷检测方法，其特征在于，包括：
获取待检测焊缝的图像；
将所述图像输入特征提取网络，提取多尺度特征；
将所述多尺度特征输入缺陷分类器，输出缺陷类型和置信度；
将所述置信度与预设阈值进行比较，输出检测结论。
```

## 从属权利要求

### 格式
```
根据权利要求[X]所述的[名称]，其特征在于，[附加技术特征]。
```

### 多项引用（合并引用）
```
根据权利要求[X]或[Y]所述的[名称]，其特征在于，[附加特征]。
```
> ⚠️ 多项引用的从属权利要求不能再被多项引用的权利要求引用（防止组合爆炸）

### 从属权利要求内容方向
1. **优选数值范围**：将独立权利要求中的宽泛范围缩小为优选范围
2. **优选结构/步骤**：指定独立权利要求中功能性特征的具体实现
3. **附加功能模块**：增加可选的增强功能
4. **材料/参数选择**：指定最优材料或工艺参数

## 布局策略

### 标准"伞形"布局（推荐）
```
权利要求1：最宽独立权利要求（产品）
├── 权利要求2：优选结构A
├── 权利要求3：优选结构B
│   └── 权利要求4：结构B的进一步限定
└── 权利要求5：优选参数范围
权利要求6：方法独立权利要求
├── 权利要求7：方法步骤优选
└── 权利要求8：方法参数优选
权利要求9：系统独立权利要求（如适用）
```

### 多维度保护建议
同一发明，尽量从多个角度撰写独立权利要求：
- 产品（装置/设备/系统）
- 方法（制造方法/使用方法/控制方法）
- 存储介质（计算机程序相关）
- 用途（化学/医药领域）

## 撰写模板

### 装置/系统类
```
一种[系统名称]，其特征在于，包括：
[模块1]，用于[功能1]；
[模块2]，与所述[模块1]连接，用于[功能2]；
[模块3]，根据[模块2]的输出，执行[操作]。
```

### 方法/流程类
```
一种[方法名称]，其特征在于，包括以下步骤：
[步骤S1]；
[步骤S2]；
[步骤S3]，得到[结果]。
```

### 计算机可读存储介质类
```
一种计算机可读存储介质，其上存储有计算机程序，其特征在于，
所述计算机程序被处理器执行时实现[方法名称]的步骤。
```

### 化合物/组合物类
```
一种[化合物/组合物名称]，其特征在于，[结构/成分描述]。
```

### 马库什（Markush）权利要求（化学领域）
```
一种通式（I）所示的化合物：[通式]
其中，R1选自[X、Y、Z]中的一种；
R2为[描述]。
```

## 语言规范

### 必须使用精确词汇
| 场景 | 推荐用词 | 禁用词 |
|------|----------|--------|
| 连接关系 | 与…连接、与…耦合、与…通信连接 | 接近、大概连接 |
| 数值范围 | 1mm~10mm、不小于5% | 约1mm、较小的、合适的 |
| 包含关系 | 包括、包含（开放式）；由…组成（封闭式） | 含有某些、等等 |
| 选择关系 | 选自A、B、C中的至少一种 | 或类似物 |

### 权利要求用语一致性
- 权利要求中使用的名称必须与说明书中保持完全一致
- 同一技术特征不得在不同权利要求中使用不同名称

### 引用关系规范
- 从属权利要求引用编号必须小于自身编号（不得向后引用）
- 独立权利要求不得引用其他权利要求
- 引用格式固定：`根据权利要求X所述的…`，不得写成"如权利要求X中的…"

### 权利要求结尾
- **每项权利要求只能在末尾使用一个句号**
- 中间各技术特征之间用分号（；）或逗号（，）分隔
- 括号使用：仅附图标记、化学式/数学式中允许括号；其他情况尽量避免

### 引用附图标记
权利要求中可以引用附图标记，但标记须放在对应技术特征后的括号内，且不限制权利要求的范围：
```
其特征在于，所述装置包括驱动模块（10）和控制单元（20）。
```

## 功能性限定注意事项

中国专利实践中，功能性限定的保护范围解释：
- **在权利要求书中**：功能性限定应解释为说明书中描述的实现该功能的具体方式及其等同方式
- **建议**：若使用功能性限定，说明书中须详细描述具体实现结构，避免保护范围过窄

### 不允许的权利要求主题（发明专利）
- 科学发现
- 智力活动的规则和方法（纯商业规则，无技术特征）
- 疾病的诊断和治疗方法（但医疗器械、药物本身可以）
- 动植物品种（育种方法可以）
- 原子核变换方法
- 平面印刷品的设计（外观设计保护）
